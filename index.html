GuildOne Inc.

Software Developer
April 2018 - Present

GuildOne Inc. is a Calgary software company that provides advanced data solutions and business intelligence to oil and gas companies. In 2018, GuildOne conducted the world’s first blockchain oil and gas royalty transaction on Energy Block Exchange (EBX), its blockchain business network for energy transactions.

GuildOne has allowed me to be part of many products such as Royalty Studio, SynMine and EBX. Being part of so many projects has allowed me to touch many parts of the programming world such as

  • OOP and functional programming
  • Unit tests
  • API testing
  • UI testing
  • Front end development
  • Blockchain development
  • Documentation

At GuildOne, I primary work on EBX, the world's first O&G royalty transaction. During my time on EBX, I was able to get Corda certified! This meant that I have a solid knowledge of blockchain concepts such as Blocks, Transactions, Consensus, UTXO, Smart contracts, Public key cryptography. As well, I am able to contribute to writing Corda code and developing Corda applications. I learn a lot while working on EBX as it has many different moving parts. I work primarily on the front end and the middle layer where the API is laid before talking to the blockchain layer.

The front end is primarily done in JavaFX, Oracle's software platform for delivering desktop applications. In the application, I help in creating a better format to render data visually by redesigning the user interface while increasing functionality for sorting tables.

Another new tool in my arsenal was being exposed to QA testing. I was part of a small team that tested the API where we did numerous types of testing such as validation testing, functional testing, runtime/error detection.

While working on EBX, I also initiate in the implementation a new web interface for EBX. Previously, EBX was available on a desktop application. Since EBX was written in the Scala programming language, I decided to use Scala.js. Scala.js is a compiler that compiles Scala source code to equivalent JavaScript code. With this ability to convert Scala code to JavaScript, we were able to use React as our front end. This is my first exposure to React, and so far, no wonder why it's one of JavaScript's most popular front end frameworks!

During my time at GuildOne, I also help develop an internal documentation site where GuildOne employees could go and look up either business related information about a product, to technical documentation about API endpoints to blockchain concepts used. This was a great opportunity for me to learn more about pipelines that are used in BitBucket and Docker Hub.
